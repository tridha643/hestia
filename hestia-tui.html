hestia · TUI design spec

Agents drive everything through the CLI; the TUI is the human's live window onto the same session state — hunk's exact model. Primitive-first, keyboard-first, one stream.

Design stance (what we take from hunk)

Hunk's TUI works because it is a projection of primitives, not a dashboard: one navigable stream (files → hunks → comments), paired prev/next keys per primitive level (, . files, [ ] hunks, { } comments), single-letter toggles, layout modes 1/2/0 (split/stack/auto), a menu bar for mouse users, and — critically — every TUI action has a CLI twin operating on the same live session, so agent actions appear in the UI the moment they happen. Hestia adopts all five rules. The TUI never has a capability the CLI lacks, and vice versa.

Primitives

stack

One worktree's whole environment. Identity = branch. Holds a slot (or a queue position). States: queued · starting · up · degraded · stopped.

, / . prev / next stack   u / d up / down   Enter focus

service

A unit inside a stack, tagged by backend: docker · proc (wrangler, vite) · tunnel. Has health, endpoint, resources.

[ / ] prev / next service   R restart   s shell   l focus logs

endpoint

How you reach a service: named URL (web.salem.modem.localhost), public URL (via hestiad's tunnel), or TCP port + injected env (DATABASE_URL).

o open in browser   y yank URL   Y yank env block   t expose / unexpose public

event

An inbound callback at the canonical URL, journaled with its routing decision (matched / affinity / broadcast / primary). Replayable into any stack.

{ / } prev / next event   e events view   r replay to selected stack

slot

A concurrency lease — 5 in v1. The queue is visible, ordered, and steerable; a stack without a slot renders exactly where it will run next.

+ / - reprioritize queued stack   K revoke slot (with confirm)

log line

Attributed to stack + service + actor (human or agent). One merged, filterable stream — per stack or fleet-wide.

/ filter   f follow   L fleet-wide log mode   g / G top / bottom

Deliberately not in the TUI

One theme: github-dark-default (hunk's default), no theme selector in v1 — the palette above is the spec. No editing of hestia.yml in-app, no metrics dashboards, no container inspector (that's lazydocker's job — s drops you into a shell instead), no multi-repo view in v1. Hunk stays sharp by refusing to become a git client; hestia stays sharp by refusing to become Docker Desktop. The TUI answers exactly three questions: what's running, how do I reach it, and what just happened — everything else is one keypress into a shell or one CLI call away.
